# 桌面宠物应用图标设计
## 目标
为 macOS 桌面宠物应用设计一枚可用于 Dock、Finder 和启动器的桌面图标。图标需要比现有概念草图更精致，同时保留“小猫陪伴”的亲近感和应用级的系统感。
## 最终方向
采用雾白灰蓝的 macOS 精修小猫头像。主体是一枚柔和圆角方形底座，使用白色到灰蓝的轻微渐变、细腻高光和柔和投影，让图标在 macOS 桌面环境中显得干净、成熟、耐看。
小猫居中展示，使用白灰毛色、浅灰蓝阴影、黑色圆眼、淡粉鼻尖和简洁胡须。表情保持亲近、安静、有一点灵气，不做眨眼、星星、薄荷青或暖奶油色装饰。
额头加入细弧形 M 毛流作为记忆点。M 需要像自然猫毛纹路，而不是硬写上去的字母；线条应更细、更软、有弧度，远看能形成 Mainey 的品牌识别，近看仍然像猫脸的一部分。
## 视觉组件
- 背景底座：macOS 风圆角方形，雾白到灰蓝渐变，带轻微内高光和外投影。
- 小猫头部：居中偏大，耳朵和脸部具备轻微立体感，边缘清楚但不粗黑。
- 额头纹：三条细弧组成的自然 M 毛流，使用低饱和灰蓝，避免粗重和机械感。
- 五官：黑色圆眼带小高光，淡粉鼻尖，简洁水平胡须，整体保持小尺寸可读。
## 产物要求
- 生成一张 1024x1024 高分辨率主图，画面留出安全边距。
- 图标应能进一步导出为 macOS app icon 需要的多尺寸资源。
- 实现计划需要根据当前 Swift/macOS 项目结构确定最终资源位置，可以使用应用资源目录或新增 AppIcon 资源结构。
## 验收标准
- 1024px 下能看到精致材质、耳朵、脸部阴影和细弧 M 毛流。
- 128px 下小猫脸和 M 额纹仍然可辨识。
- 32px 下至少能识别为白灰小猫头像，而不是普通灰色圆形或抽象图案。
- 整体不使用薄荷青作为主色，不偏暖奶油玩具感，不像硬贴字母 M。
## 实现备注
实现阶段可以先生成位图图标，再按 Swift/macOS 项目结构接入。接入时需要避免改动现有宠物 sprite sheet 资源和当前未提交的应用逻辑变更。
